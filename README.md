# 🔍 Smart Doc Search
### AI-Powered Semantic Document Search Engine

**Resume-worthy full-stack project** using Node.js · TypeScript · Express · MongoDB · Pinecone

---

## 🧠 What This Project Does

A semantic document search engine. Unlike keyword search (which only matches exact words), this searches by **meaning**.

| Keyword Search | Semantic Search (this project) |
|---|---|
| "dog food" → finds "dog food" | "dog food" → also finds "puppy meal", "pet nutrition" |
| "artificial intelligence" → only that phrase | "artificial intelligence" → finds "neural networks", "deep learning", "ML" |

**How it works:**
1. Documents are converted to 384-number vectors ("embeddings") by a local AI model
2. Vectors are stored in Pinecone (vector database)
3. When you search, your query becomes a vector → Pinecone finds the closest document vectors → results ranked by similarity

## 🚀 Quick Start

### Step 1: Get Free Credentials

**MongoDB** (choose one):
- Local: install MongoDB → `mongodb://localhost:27017/smart-doc-search`
- Cloud: [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) → Free M0 cluster

**Pinecone** (free, no credit card):
1. Sign up at [pinecone.io](https://www.pinecone.io)
2. Go to API Keys → copy your key

---

### Step 2: Setup

```bash
# Install dependencies
npm install

# Copy env template and fill in your values
cp .env.example .env
# Edit .env: add MONGODB_URI and PINECONE_API_KEY
```

---

### Step 3: Run

```bash
# Start the server (dev mode with auto-restart)
npm run dev

# In another terminal, seed sample data
npm run seed
```

Open http://localhost:3000 in your browser.

### Search Parameters
```
GET /api/documents/search?q=machine+learning&category=technology&limit=5
```
- `q` — search query (required)
- `category` — filter by category (optional)
- `limit` — number of results, max 20 (default: 5)

## 💡 Key Concepts (for interviews)

**Q: Why Pinecone instead of MongoDB for search?**
MongoDB can store vectors but comparing every document for similarity is O(n) — slow. Pinecone uses ANN (Approximate Nearest Neighbor) algorithms to find similar vectors in O(log n), milliseconds even with millions of vectors.

**Q: What is an embedding?**
A list of numbers (vector) representing the semantic meaning of text. Generated by an AI model. Similar meanings = similar vectors. We use `all-MiniLM-L6-v2` (384 dimensions, runs locally, free).

**Q: How is MongoDB + Pinecone split?**
MongoDB = source of truth (full content, all fields). Pinecone = search index (vector + small metadata + MongoDB ID). Search finds IDs in Pinecone → fetch full content from MongoDB.
